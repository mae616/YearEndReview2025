# TASK-0-2: デザインの視覚的整合性チェック（Sprint 0）

## 目的/DoD（Definition of Done）

既存スライド（slides.md）のデザインが、AGENTS.mdで定義された以下の指針に準拠しているかを確認し、必要に応じて微調整を行う。

**完了条件**:
- [ ] 色のコントラストが十分に確保されている（文字が明瞭に読める）
- [ ] 余白が適切（意図のない大きな余白がない）
- [ ] 強調箇所が明確（色・枠で重要箇所が一目で分かる）
- [ ] トランジションが全体で統一されている（slide-left）
- [ ] 色使いが統一され、意味・構造・視線誘導に寄与している
- [ ] **デザインが洗練され、エンジニア感が軽減されている**（NEW）
- [ ] **用語が「前期/中期/後期」に全スライドで統一されている**（NEW）
- [ ] チェック結果を記録し、改善点があれば修正完了

### ステップ2: 視覚的整合性チェックリスト

各スライドを順番に確認し、以下の項目をチェックする：

#### 2-1. 色のコントラスト
```
目的: 文字が十分に読めるコントラストが確保されているか

チェック項目:
□ タイトル文字（h1, h2）が背景に対して明瞭に読める
□ 本文テキスト（text-gray-600, text-gray-700, text-gray-800）が読みやすい
□ 強調色（text-blue-600, text-green-700, text-purple-700等）が背景に対して十分なコントラスト
□ 背景色（bg-*-50系）に対する文字色が適切
□ タイトルスライドの背景画像に対する文字色（#565656）が読みやすい

参照: AGENTS.md §Color / Contrast
基準: WCAG 2.1 AA基準（コントラスト比 4.5:1 以上が望ましい）
```

#### 2-2. 余白の適切さ
```
目的: 意図のない大きな余白がないか、情報密度が適切か

チェック項目:
□ スライド上下左右の余白が適度（感じられる程度）
□ 各セクション間の余白（mt-*, gap-*）が統一されている
□ 意図せず大きな空白がないか（grid, flex レイアウトの確認）
□ 情報が一目で把握できる量に収まっているか

参照: AGENTS.md §Layout / Information Density
```

#### 2-3. 強調箇所の明確さ
```
目的: パッと見て「どこが重要か」「何を見ればいいか」が分かるか

チェック項目:
□ 重要な単語が色（font-bold, text-*-600/700）で強調されている
□ ボックス（border-l-4, bg-*-50）で構造が明確
□ 視線が自然に重要箇所に誘導されるレイアウトになっているか
□ 各時期（第1期/第2期/第3期）の色分けが一目で分かるか
  - 第1期: 青・赤・黄
  - 第2期: 緑
  - 第3期: 紫・インディゴ

参照: AGENTS.md §Visual Expression
```

#### 2-4. トランジションの統一性
```
目的: ページ切り替えが全体で統一され、聞き手が酔わないか

チェック項目:
□ frontmatterで `transition: slide-left` が設定されている（全体共通）
□ 個別スライドでトランジションが上書きされていないか
□ ページめくりがスムーズで、ふんわりとした印象か
□ 強いアニメーションや派手な演出がないか

参照: AGENTS.md §Animation & Transition
確認方法: キーボード矢印キーでスライドを送り、切り替えアニメーションを観察
```

#### 2-5. 色使いの統一性
```
目的: スライド全体で色が統一され、意味・構造に寄与しているか

チェック項目:
□ 各時期のテーマカラーが一貫している
  - 第1期: 青（bg-blue-50）、赤（bg-red-50）、黄（bg-yellow-50）
  - 第2期: 緑（bg-green-50）
  - 第3期: 紫・インディゴ（bg-purple-50, bg-indigo-50）
□ 同じ意味を持つ要素が同じ色で表現されている
□ 色が装飾ではなく、意味・構造・視線誘導のために使われている
□ 落ち着いたトーン、くすみ系の色合いになっているか（Tailwind *-50系）

参照: AGENTS.md §Typography & Color
```

#### 2-6. デザインの洗練度（NEW）
```
目的: シンプルすぎる・エンジニア感が強いデザインを改善

チェック項目:
□ ボックスデザインが単調すぎないか（改善案: アイコン追加、グラデーション、影など）
□ 「今日話すこと」スライドの3つのボックスが洗練されているか
  - 現状: シンプルなborder-l-4 + bg-*-50
  - 改善案: アイコン追加、グラデーション背景、shadow効果、角丸調整など
□ 各時期のセクションタイトルスライドが印象的か
□ リストやボックスに視覚的な工夫があるか（アイコン、番号装飾など）

参照: AGENTS.md §Visual Expression, §Typography & Color
改善指針: 「緩くデザインのいいデザイン」「落ち着いたトーン、くすみ系」を維持しつつ洗練

具体的な改善案:
- アイコン追加（Font Awesome、Heroicons等のシンプルなアイコン）
- グラデーション背景（bg-gradient-to-r from-*-50 to-*-100）
- 影効果（shadow-md, shadow-lg）
- ホバー効果（プレゼン時は不要だが、視覚的な立体感に寄与）
- 角丸調整（rounded-lg → rounded-xl）
- border装飾の工夫（border-l-4 → border-l-8 + グラデーション）
```

#### 2-7. 用語統一（NEW）
```
目的: 「第1期/第2期/第3期」を「前期/中期/後期」に全スライドで統一

チェック項目:
□ 「今日話すこと」スライド: 第1期 → 前期、第2期 → 中期、第3期 → 後期
□ セクションタイトルスライド: 第1期 → 前期、第2期 → 中期、第3期 → 後期
□ 各セクションの見出し: 「第○期の特徴」→「○期の特徴」
□ 本文中の言及: すべて統一されているか
□ Speaker Notes: 用語が統一されているか

修正対象ファイル: slides.md
修正方法: 検索置換（第1期 → 前期、第2期 → 中期、第3期 → 後期）

注意: RDD（doc/rdd.md）には「第1期」等の表記があるが、
スライド本体では「前期/中期/後期」に統一する（ユーザー要望）
```

## 難所検証方針

このタスクは**既存スライドの視覚的確認**のため、新規実装はない。
最小サンプルは不要。

もし、カスタムCSS追加やレイアウト変更が必要になった場合は、
以下の方針で最小検証を行う：

1. **単一スライドでの検証**:
   ```markdown
   ---
   layout: default
   ---

   # テストスライド

   <div class="p-5 bg-purple-50 rounded-lg border-l-4 border-purple-500">
     <div class="font-bold text-lg text-gray-800">テストテキスト</div>
   </div>
   ```

2. **styles.cssでの検証**:
   ```css
   /* 追加したスタイルの動作確認 */
   .test-class {
       color: #565656;
   }
   ```

3. **検証後は削除可**（コメントに記載）

**Alternatives（代替案）**:
- デフォルトフォントのまま（現状維持）
- システムフォントを指定（font-family: sans-serif）

**Status（状態）**:
提案中（ユーザー承認待ち）
